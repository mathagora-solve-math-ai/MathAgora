### 02.11

1. **방향성 구체화**
    - 큰 목적: 콴다보다 신뢰성 있게 → 다양한 LLM 풀이 비교 시각화
    - 시스템 흐름
        - 시험지 이미지 촬영 (또는 업로드)
        - 문제 단위 detection
            - 사용자가 특정 문제를 선택
        - 선택된 문제를 여러 LLM이 동시에 풀이
            - 이때 step의 단위는 LLM에게 ‘풀이의 흐름이 바뀌는 대로 구분하라’ 정도로 기준을 제공해서 알아서 나뉘게 함
        - Flow Map Generator
            - 에이전트
            - LLM이 나눈 step 대로 flow map을 만드는 역할
            - Step 별로 유사한 그룹 생성 및 그룹 이름 작명 (소제목보다 조금 큰 중제목 정도의 느낌으로)
            - 전체 flow map 설계
        - Flow Map 시각화
            - Generator가 만든 flow map 시각화
2. **향후 계획**
- 인범: DLA(문제 단위 detection) 모듈 설계
- 은빈: 프론트 및 기본 기능 설계
- 고운: Flow Map Generator 설계
- 금주 금요일(2/13)까지 1차 서비스 완성 목표
